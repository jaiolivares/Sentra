Les comentaré un poco mi experiencia y a lo que me vi enfrentado al realizar este ejercicio técnico.

  • Utilicé Angular y AngularMaterial en sus versiones 18.
  • Utilicé una Api pública que tiene datos de prueba de "productos" para emular la respuesta de un BackEnd https://fakestoreapi.com/
  • El menú es responsivo por lo tanto tiene su manejo propio para visualizarlo en resoluciones menores.
  • Cree un Administrador de productos CRUD aplicando el manejo de AngularMaterial para todos los elementos.
  • Me faltó realizar validaciones para no permitir registros duplicados.
  • Me faltó entregar un mensaje de confirmación al grabar, eliminar o editar los datos.
  • Algunas de las características que le agregue a la grilla fueron:
    • Filtro por Categoría que permite seleccionar una o más.
    • Textbox para realizar búsqueda por todos los campos de la grilla.
    • Posibilidad de ordenar los registros haciendo click en sus correspondientes cabeceras.
    • Seleccionar la cantidad de registros que se mostrarán por página, por defecto parte en 5.
    • Paginador.
    • Algunos ToolTips de ayuda.
    • Al momento de crear o editar se puede cargar una URL de una imagen que se mostrará en el formulario, la idea es que se cargara una imagen desde el PC del usuario, pero como estoy trabajando con una Api de pruebas, no puedo guardar en BD las rutas finales de las imágenes.
